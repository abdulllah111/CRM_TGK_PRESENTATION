Загрузка данных

Прежде чем отвечать, система должна "переварить" информацию. Это фундамент системы.

01. Source
Load & Clean
Извлечение текста из PDF, таблиц и баз. Удаление шума и лишней разметки.
02. Split
Chunking
Разбиение текста на "смыслы" — небольшие блоки, которые удобно сравнивать.
03. Vectors
Embedding
Превращение слов в математические векторы в многомерном пространстве.

Умная индексация

Мы не просто сохраняем текст, мы создаем "умную" базу ответов, генерируя вопросы для каждого фрагмента.

01. Processing
Smart Chunking
Разрезаем файлы на смысловые чанки по 1000 токенов для точного попадания.
02. Q-Gen
7 Вопросов
Для каждого чанка LLM создает 7 потенциальных вопросов, которые может задать пользователь.
03. Index
Q-Embeddings
Ембедятся именно вопросы — так поиск "вопрос-к-вопросу" работает намного точнее.

Входная точка

Пользователь что-то написал. Первым делом нужно понять, что с этим делать.

"Привет! Сколько стоит доставка в Казань и какие сроки?"

Сырой запрос может быть неточным, с опечатками или слишком сложным.

Глубокое понимание

Если выбран поиск, мы превращаем "кривой" запрос в идеальный ключ для базы знаний.

Rewrite
Улучшение

Intent
Тематика

Expansion
Синонимы
Генерация вариаций вопроса для максимального охвата при поиске.

Улучшение вопроса

Люди формулируют мысли путано. ИИ переписывает вопрос так, чтобы машине было легче искать.

"Скока везти до Казани?" (Оригинал)
Query Expansion (Расширение):
  • 1. "Тарифы на доставку грузов в Казань"
  • 2. "Сроки транспортировки в Республику Татарстан"
  • 3. "Стоимость логистических услуг регион Казань"

Умный поиск

Мы находим десятки похожих фрагментов, но до ИИ дойдут только самые лучшие.

Retrieval
Быстрый поиск Top-20 кандидатов по близости векторов вопросов.
Reranking
Кросс-кодировщик перечитывает Top-20 и выбирает 3 самых точных чанка.

Реранкинг (Rerank)

Поиск дает количество, Реранкер дает качество. Мы отбираем жемчужины.

SMART FILTER

Топ-3 Лучших

Cross-Encoder модель перечитывает кандидатов и оставляет только те, которые 100% отвечают на вопрос.

Рождение ответа

Финальный аккорд: сборка всех найденных пазлов в один логичный ответ.

[SYSTEM PROMPT]
Ответь пользователю, используя только предоставленные факты.
[CONTEXT: Факты о ценах и сроках в Казани...]
[USER: "Сколько стоит доставка в Казань?"]

Ответ сформирован

"На основании внутренней базы знаний: доставка в Казань стоит 500 рублей и занимает 3 рабочих дня."
